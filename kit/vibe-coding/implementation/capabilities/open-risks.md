# Open Risks

Risks that still need implementation design:

- actor identity mapping;
- secret management for trusted and fork PR contexts;
- prompt injection through untrusted text;
- monorepo instruction scope;
- generated file policy;
- visible human override;
- bounded AI review loops;
- false confidence from tests;
- unsupported uncertainty language.

Each risk needs a deterministic check, explicit receipt gate, or documented fallback.
