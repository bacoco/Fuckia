# PR Template

Every PR must include:

```text
Linear issue:
Linear active spec/doc:
Scope:
Allowed files:
Forbidden files:
Destructive-change guard:
Existing pipeline preserved:
Verification:
Archive snapshot:
Residual risks:
```

Risky PRs also include:

```text
Plan issue:
Plan-review issue:
Plan reviewer:
Code reviewer:
Verification issue:
```
